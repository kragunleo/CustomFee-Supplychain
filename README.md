# CustomFee-Supplychain
Customs Brokerage Fee Audit Analytics
Project Overview
In the logistics industry, import operations are governed by specific regulatory fee scales. This project was developed to audit the alignment between invoiced amounts from freight forwarders and the official HAD FEDERMAR fee structures in Ivory Coast.

The goal was to build a repeatable, data-driven control framework that identifies financial variances and documentation gaps, ensuring that all third-party billing follows standardized regulatory benchmarks.

üéØ Key Objectives
Recomputation: Build a Python engine to calculate expected Customer Broker Fees (CBF) based on customs value and regulatory slices.

Compliance Validation: Verify fixed-fee rubrics (such as administrative visas) against known legal flat rates.

Anomaly Detection: Automatically flag dossiers where the variance between billed and theoretical amounts exceeds a 5% threshold.

Remote Workflow: Establish a digitized audit trail capable of processing cross-border logistical data.
